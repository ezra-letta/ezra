## THE PROBLEM

### Section headline
**Every other coding tool forgets you.**

Claude Code, Cursor, Copilot -- they all start from zero every session. Close the terminal, lose the context. Switch machines, lose the context. Come back tomorrow, re-explain everything.

The longer you use them, the more time you waste on setup instead of building.

---

## THE SOLUTION (3 pillars)

### 1. Memory that compounds

Your agent builds a knowledge base about your project with every session. Conventions, patterns, gotchas, preferences -- all learned through working together, not configured upfront.

Day 1: capable coding agent.
Day 30: a teammate with context a new hire would need weeks to build.

**[Visual: Side-by-side of empty memory on day 1 vs. rich memory on day 30]**

---

### 2. Your agent, everywhere

Same agent from your terminal, browser, or phone. Debug at your desk with the CLI. Check in from your phone via Telegram. Review in the browser through the ADE. One agent, one brain, every interface.

**[Visual: Terminal -> Phone -> Browser, same agent responding with context]**

---

### 3. Transparent and model-agnostic

Your agent's memory is git-backed markdown files you can read, edit, and version-control. No black box. No lock-in.

Switch between Claude, GPT, and Gemini with a single command. Your agent's knowledge transfers instantly because it lives in the memory system, not in any model's conversation window.

**[Visual: `cat memory/system/project.md` showing learned knowledge + `/model` command switching providers]**

## HOW IT WORKS

**Step 1:** Install and start
```
npm install -g @letta-ai/letta-code
letta
```

**Step 2:** Code like normal. Your agent reads files, writes code, runs tests, executes commands.

**Step 3:** Keep using it. The agent automatically builds memory about your project, preferences, and patterns.

**Step 4:** Come back tomorrow. Everything it learned is still there. No re-explaining.
