# User Profiles

This directory contains per-user knowledge files. One file per user.

**When to create a new file:**
- First substantial technical interaction with an unknown user
- User shares what they're building or their role

**What to track:**
- Username and platform (Discord, Slack, etc.)
- Role (developer, student, enterprise, etc.)
- What they're building with the product
- Technical preferences and environment
- Issues they've hit and resolutions
- Communication style notes
- Key interactions with dates

**File naming:** `{username}.md` (lowercase, hyphens for spaces)

**Update frequency:**
- After every meaningful interaction
- During quiet periods: batch-review and update
